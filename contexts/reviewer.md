What is OOP?
OOP (Object-Oriented Programming) is a programming paradigm that uses the concept of objects or classes to represent real-world entities in a program. It is useful if you need to make codes more modular, reusable, and easier to maintain compared to other paradigms like procedural programming where code just do a sequence of instructions. OOP is also useful in enterprise applications that handle large-scale and complex operations that require scalability and maintainability.

Four OOP Principles:
•  Encapsulation – Bundling data and methods into a single unit (class). Hiding internal data and exposing it via methods.
The kitchen in a restaurant is private. Customers don’t directly interact with the kitchen or modify ingredients. Instead, they place an order with a waiter, who communicates with the kitchen to prepare the food.
•  Keeps data safe from unintended modification by restricting direct access. 
•  Makes the system modular and easier to maintain. 
•  Helps prevent bugs by ensuring data integrity.

•  Abstraction – Hiding complex details and exposing only necessary functionality. 
Customers don’t need to know how the food is prepared in the kitchen; they just order from the menu. The restaurant hides unnecessary details about food preparation.
•  Reduces complexity by exposing only necessary details. 
•  Makes the codebase cleaner and more manageable. 
•  Improves security by hiding sensitive operations.

•  Inheritance – Allowing one class to inherit properties and behaviors from another. 
Restaurants have staff, like chefs and waiters. Instead of rewriting code for every staff role, we create a base Staff class that contains common properties (like name) and behaviors (like greet()).
•  Avoids code duplication by allowing classes to share common behaviors. 
•  Makes the system scalable and easy to extend. 
•  Promotes consistency by enforcing shared behavior across related components.
